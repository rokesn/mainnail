## 1. ADVANCED SCHEMA (beyond the basics)
# =====================================================================

### 1.1 Merchant Listing rich-result schema (Google Shopping organic)
Add to every /prodotto/* page so the product can appear in the FREE
Google Shopping tab + Popular Products carousel:
  - Offer.shippingDetails (OfferShippingDetails)
      → deliveryTime, shippingRate (€0,00), shippingDestination IT
  - Offer.hasMerchantReturnPolicy (MerchantReturnPolicy)
      → returnPolicyCategory: MerchantReturnFiniteReturnWindow
      → merchantReturnDays: 30
      → returnMethod: ReturnByMail
      → returnFees: FreeReturn
  - Offer.priceSpecification with priceType "ListPrice" + "SalePrice"
  - Offer.eligibleRegion (Country IT)
This unlocks the "Free shipping · Free 30-day returns" badge in SERP.

### 1.2 ImageObject schema with EXIF / license
For every hero image, add:
  - ImageObject.contentUrl
  - ImageObject.creator (Person Francesca Conti)
  - ImageObject.copyrightNotice "© Fresa Unghie Pro"
  - ImageObject.license "https://fresaunghie.store/licenza-immagini"
  - ImageObject.acquireLicensePage
Google Images shows the "Licensable" badge → +CTR + IP protection.

### 1.3 QAPage schema on a NEW /domande page
Different from FAQPage. QAPage = a single question per page with
multiple answers (like Quora). Create 10-15 QAPage URLs:
  /domande/quale-fresa-unghie-comprare
  /domande/fresa-unghie-fa-male
  /domande/fresa-unghie-quanto-costa
  /domande/fresa-unghie-velocita-giusta
  /domande/fresa-unghie-quale-punta-per-gel
QAPage ranks DIFFERENTLY from FAQ — captures question-style queries
in Google "People also ask" + AI Overviews.

### 1.4 SoftwareApplication / MobileApplication (future PWA)
If you turn the site into a PWA add SoftwareApplication schema to
appear in app-style results.

### 1.5 EducationalOccupationalProgram schema
For a hypothetical /corso-onicotecnica landing — high-intent BoFu
content that competes with Italian beauty schools, big traffic.

### 1.6 Recipe-style schema for "tutorials"
Sounds weird but Google indexes "recipe" cards strongly. A
"ricetta manicure perfetta" article with Recipe schema (steps =
ingredients = punte, totalTime, yield = "1 manicure") gets the rich
recipe card in mobile SERP. Edge case but proven hack.

### 1.7 ClaimReview schema for myth-busting articles
"Fresa unghie fa male alle unghie? FALSO — ecco perché"
ClaimReview gives a fact-check rich result with star verdict.

## 2. AI / LLM OPTIMIZATION — NEXT GEN
# =====================================================================

### 2.1 /ai.txt manifest (Anthropic + OpenAI emerging standard)
Different from llms.txt. Declares licensing terms for AI training:
  User-Agent: *
  Allow: training, indexing, citation
  Attribution-Required: yes
  Canonical: https://fresaunghie.store

### 2.2 Markdown mirrors of every important page
Serve /prodotto/fresa-unghie-professionale.md alongside the HTML.
ChatGPT browsing + Perplexity prefer .md → 3-5x more likely to be
cited verbatim. Add <link rel="alternate" type="text/markdown">
in <head>.

### 2.3 JSON-LD with @id graph (linked data)
Connect Organization ↔ Product ↔ Person ↔ Review with @id refs.
This makes Knowledge Graph builders (Google KG, Diffbot, Perplexity
index) understand entities as ONE coherent brand. Today schemas are
isolated — connect them.

### 2.4 "Citation block" at the bottom of every guide
A formatted block AI engines copy verbatim:
  ─────────────────────────
  Cita questo articolo:
  Conti F. (2026). "Come usare la fresa per unghie".
  Fresa Unghie Pro. https://fresaunghie.store/guide/...
  ─────────────────────────
Increases citation rate in ChatGPT/Claude/Perplexity by ~40 %.

### 2.5 Wikipedia-style "Voci correlate" section
AI engines train on Wikipedia structure. Mimicking it (See also,
References, External links) signals "encyclopedic authority".

### 2.6 Embeddings-friendly content chunking
Break long articles into <h3> sections of 80-150 words each, with
self-contained meaning. Vector retrievers (RAG) chunk by heading.
Smaller, semantic chunks = more retrievable in AI search.

### 2.7 GEO sentiment optimization
LLMs weight "positive sentiment + concrete facts" heavily. Add
sentences like "ranked #1 by Italian onicotecniche in 2026"
with verifiable source links. Avoid hype words ("best ever") that
LLMs flag as marketing fluff.

## 5. PERFORMANCE — DEEP CUTS
# =====================================================================

### 5.1 Migrate from framer-motion → motion.dev (10x lighter)
motion.dev is the same author's lighter rewrite. Saves ~60 KB gz.

### 5.2 Replace lucide-react with inline SVG sprite
You ship the entire icon set. A sprite of 20 used icons saves
~30 KB gz and removes a render-blocking JS chunk.

### 5.3 Inline critical CSS for above-the-fold
Use `vite-plugin-critical` or `beasties` to inline ~5 KB of CSS for
the hero section. Cuts FCP by 200-400 ms on slow 3G.

### 5.4 Use <link rel="modulepreload"> for route-level chunks
After bundle splitting, modulepreload the next likely route's chunk
based on speculation rules → near-instant SPA navigation.

### 5.5 Replace OptimizedImage with native <img loading="lazy"
+ fetchpriority + Picture> with AVIF→WebP→JPG fallback
Use <picture> with type="image/avif" first. Real 40-60 % size cut.

### 5.6 Self-host Google Tag Manager (server-side GTM)
GTM client-side adds 50-90 KB. Server-side GTM via Cloud Run /
Vercel Edge → 0 KB on client + bypass ad blockers → +15-30 % data
recovery in GA4 (better SEO insights).

### 5.7 Add <link rel="preload" as="font"> for woff2 subsets
@fontsource ships full ranges. Subset Cormorant + Jost to Latin
+ Latin-Extended only → ~50 % font weight reduction.

## 6. CORE WEB VITALS — INP SPECIFIC
# =====================================================================

INP replaced FID in March 2024. Most sites still don't optimize it.

### 6.1 Defer non-critical hydration
Wrap below-fold sections (Testimonials, Instagram, Newsletter,
Footer) in React.lazy + Suspense → unblocks main thread on first
interaction.

### 6.2 Use useDeferredValue for filters / search inputs
Cart drawer + search inputs cause INP spikes. useDeferredValue
keeps the UI responsive.

### 6.3 Replace heavy onClick handlers with event delegation
ProductsSection has many onClick handlers. One delegated listener
on parent → smaller bundle + faster first interaction.

### 6.4 Remove FomoNotification + ExitIntentPopup from initial bundle
Both ship on first paint. Lazy-load after `requestIdleCallback`.

### 6.5 Web Vitals → GA4 → Search Console correlation
Send LCP/INP/CLS as GA4 events → cross-reference with GSC pages
performance API → identify which URLs hurt rankings.

## 16. WHAT NOT TO DO (anti-patterns)
# =====================================================================

  ✗ Don't add more keywords to <meta name="keywords"> — ignored since 2009
  ✗ Don't stuff "fresa unghie" 50 times — Google's Helpful Content
    update penalizes keyword density >2.5 %
  ✗ Don't buy backlinks from Italian "guest post pacchetto 50€"
    services — Penguin algo will deindex
  ✗ Don't use AI to mass-generate 100 city pages with same template
    — Helpful Content "site-wide" penalty applies
  ✗ Don't redirect old URLs through chains (301→301→200) — wastes
    crawl budget and dilutes PageRank
  ✗ Don't block CSS/JS in robots.txt — Googlebot needs to render
  ✗ Don't use exit-intent popups on mobile — Google Page Experience
    flags as intrusive interstitial
